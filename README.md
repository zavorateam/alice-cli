# Alice CLI & API Server

Неофициальный клиент и API‑сервер для Яндекс Алисы, работающий через обратную разработку протокола WebSocket. Позволяет общаться с Алисой из терминала или интегрировать её в любые приложения, поддерживающие OpenAI API.

## Возможности

- **CLI‑чат** – минималистичный интерактивный диалог с Алисой прямо в терминале. Один чат, одна команда `/exit` для выхода с автоматическим удалением чата.
- **API‑сервер** – FastAPI‑сервер, совместимый с эндпоинтом `/v1/chat/completions` (как у OpenAI). Управление несколькими чатами через веб‑интерфейс.
- **Автоматическое удаление** – при выходе из CLI чат удаляется с серверов Яндекса.
- **Локализация** – английский интерфейс по умолчанию, русский – опционально (`--lang ru`).
- **Безопасность** – все учётные данные хранятся только локально в файле `.alice_cookies`, полученном через авторизацию в браузере.

## Требования

- Python 3.8+
- Зависимости из `requirements.txt`:
  - `websockets` – для WebSocket‑соединения
  - `aiohttp` – для HTTP‑запроса удаления чата
  - `playwright` – для получения cookies (только в `get_cookies.py`)
  - `fastapi`, `uvicorn` – для API‑сервера

## Быстрый старт

### 1. Клонирование и установка

```bash
git clone https://github.com/your/alice-cli.git
cd alice-cli
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate.ps1 или .venv\Scripts\activate.bat
pip install -r requirements.txt
playwright install chromium
```

### 2. Получение cookies

Запустите скрипт авторизации, войдите в аккаунт Яндекса в открывшемся браузере и нажмите Enter:

```bash
python get_cookies.py
```

После этого появится файл `.alice_cookies`, необходимый для работы всех инструментов.

### 3. Запуск CLI‑чата

```bash
python alice_cli.py
```

Вы увидите приглашение `> `. Напишите что‑нибудь и получите ответ. Для выхода введите `/exit` – чат автоматически удалится.

Для локализации на русский добавьте флаг:

```bash
python alice_cli.py --lang ru
```

### 4. Запуск API‑сервера

```bash
python alice_api_server.py
```

Сервер стартует на `http://127.0.0.1:8000`. Документация доступна по адресу [`http://127.0.0.1:8000/docs`](http://127.0.0.1:8000/docs).

Пример запроса к OpenAI‑совместимому эндпоинту:

```bash
curl -X POST http://127.0.0.1:8000/v1/chat/completions \
     -H "Content-Type: application/json" \
     -d '{"messages": [{"role": "user", "content": "Привет, Алиса!"}]}'
```

В ответе вы получите стандартную структуру ответа OpenAI. Также доступны ручки:

- `GET /dialogs` – список активных чатов
- `POST /dialogs` – создать новый чат
- `DELETE /dialogs/{dialog_id}` – удалить чат

Сервером можно управлять, запустив его с флагом `--cli`, чтобы войти в интерактивный режим с командами `/new`, `/delete`, `/list`.

## 🛠 Особенности и детали реализации

- **Протокол** – клиент эмулирует веб‑версию Алисы (`alice.yandex.ru`), подключаясь к `wss://uniproxy.alice.yandex.ru/uni.ws`.
- **UUID** – уникальный идентификатор устройства автоматически извлекается из куки `alice_uuid` (если она отсутствует, используется захардкоженный запасной вариант).
- **Удаление чата** – отправляется HTTP POST на `https://rpc.alice.yandex.ru/dialog/remove_dialog`, затем обновляются подписки WebSocket.
- **Безопасность** – cookies хранятся локально и никогда не передаются сторонним серверам. Все соединения идут напрямую к Яндексу.
- **Обработка ошибок** – WebSocket автоматически переподключается при обрыве, а исключения не роняют процесс.

## 🧩 Использование в сторонних проектах

API‑сервер можно интегрировать с любым инструментом, поддерживающим OpenAI Chat Completions: Open WebUI, LangChain, Continue и т.д. Достаточно указать `base_url=http://127.0.0.1:8000/v1` и любой ключ API (не проверяется).

## ❗ Возможные проблемы

- **Ошибка `extra_headers`** – обновите `websockets` до последней версии (`pip install --upgrade websockets`), либо используйте встроенный обходной путь (присутствует в коде).
- **Не удаётся удалить чат** – установите `aiohttp`. Без него чат не будет удаляться автоматически, но CLI предупредит об этом.
- **Не появляется ответ на первое сообщение** – убедитесь, что файл `.alice_cookies` актуален, и повторите авторизацию.
